# Note Structure System

## Purpose

NoteU should not encourage users to write random notes.

Every note should help users understand, remember, and apply what they learned.

The note structure should train users to think deeper.

## Core Note Format

Each note should answer three questions:

1. What did I learn?
2. Why does this matter?
3. Where can I apply this?

## 1. What Did I Learn?

This captures the main idea.

The user explains the lesson in their own words.

Example:

I learned that delayed gratification means choosing a better future reward instead of a smaller immediate reward.

## 2. Why Does This Matter?

This captures meaning and importance.

The user explains why the idea is valuable.

Example:

This matters because many bad financial decisions happen when people choose short-term comfort over long-term stability.

## 3. Where Can I Apply This?

This connects learning to real life.

The user identifies where the lesson can be used.

Example:

I can apply this when deciding whether to spend my salary immediately or save part of it first.

## Optional Fields

- Source
- Category
- Tags
- Difficulty
- Personal reflection
- Related notes

## AI Usage

The AI should analyze all three parts.

It should detect:

- Main concept
- Learning domain
- Practical application
- Related ideas
- Review questions
- Identity signals

## Why This Matters for NoteU

This structure makes NoteU different from regular note apps.

Regular note apps ask:

What do you want to write?

NoteU asks:

What did this learning do to you?

## Product Philosophy

A good note does not only record information.

A good note reveals understanding.
