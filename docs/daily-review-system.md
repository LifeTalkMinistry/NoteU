# Daily Review System

## Purpose

The Daily Review System verifies that learning has been retained, not just recorded.

Instead of encouraging users to collect notes, NoteU encourages users to remember, understand, and connect what they have learned.

---

# Daily Goal

Every day, users complete a 30-minute learning session using only their own notes.

The session adapts based on previous performance and focuses on long-term retention.

---

# Review Flow

1. User opens Daily Review.

2. NoteU selects notes due for review.

3. Questions are generated automatically.

Examples:

- What does this concept mean?
- Explain this idea in your own words.
- Which field does this belong to?
- How does this relate to another concept?

4. User answers.

5. NoteU evaluates confidence and retention.

6. Identity Score updates.

---

# Spaced Repetition

Recently learned notes appear more often.

Well-retained notes appear less frequently.

Forgotten concepts return sooner.

---

# Review Metrics

The system tracks:

- Review completion
- Recall accuracy
- Confidence
- Long-term retention
- Consistency
- Review streak

---

# Review Principles

The goal is not perfection.

The goal is continuous improvement.

Users should never feel punished for forgetting.

Instead, forgetting simply tells NoteU what needs reinforcement.

---

# Product Philosophy

Notes store knowledge.

Reviews strengthen knowledge.

Retention transforms knowledge into identity.
